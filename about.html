What is this?

This is the open research interface for the Control Systems group at the Technical University of Sofia, Plovdiv Branch — an engineering department whose work spans classical and modern control theory, intelligent systems, robotics, and industrial automation.

The site exists to make the group's work visible and accessible. Academic research too often remains locked inside conference papers and university archives. Here, we document active projects, publish research notes, and maintain a running account of what we are building and why. Everything is written for an audience of engineers, PhD students, and technically curious visitors — not press releases.

The design intentionally reflects the aesthetic of the tools we use every day: terminals, monospace editors, oscilloscope readouts, block diagrams. If it looks like a laboratory instrument rather than a marketing page, that is by choice.

  • Research domain overview with engineering-accurate descriptions
  • Blog — academic notes, project updates, and field reports
  • Homebrew — open-source projects from the group's researchers
  • PhD student profiles and active research topics
  • IRC room — live, open-access channel for discussion
  • Contact form routing directly to the department

The website is itself a research artefact. It is built without frameworks, bundlers, or transpilers — every page is plain HTML, CSS, and JavaScript. The backend runs on a self-hosted NixOS machine behind a Cloudflare Tunnel. The source is open, the dependencies are minimal, and the stack is documented below.

Innerworkings

A full technical breakdown of every layer of the stack. All components are open-source or standard platform services. Nothing proprietary.

# ctrl-sys-site
HTML5 CSS3 Vanilla JS Node.js 22 SQLite Ergo IRCd NixOS 25.11 Netlify Cloudflare open source
Frontend
  • HTML5 — semantic markup, no templates or preprocessors
  • CSS3 — ~3200-line master sheet, 8 named themes, CSS variable system
  • Vanilla JS — no frameworks, no bundlers, no transpilers
  • grid-fx.js — 14+ cycling canvas visualisation modes
  • DOMPurify v3.2 — blog markdown sanitisation
  • Tabler Icons — SVG icon library via CDN
  • Leaflet.js — campus map, contact section
Backend
PackagePurpose
Node.js 22 + ExpressHTTP runtime and API framework
better-sqlite3Synchronous SQLite — post storage, zero-config
multerMultipart file uploads — images · PDFs · archives (50 MB cap)
express-rate-limitRate limiting — 5 req/h subscribe, 30 req/15 min admin
node:crypto AES-256-GCMSubscriber email encryption at rest
morgan · dotenv · corsDev logging, environment, cross-origin policy
IRC
  • Ergo IRCd — single Go binary, no external dependencies
  • Channel — #control-systems · open guest access · 50-message history
  • Client — irc.js, vanilla WebSocket, no IRC library
  • Nick persistence — localStorage, 7-day TTL
Hosting & Deployment
  • Netlify — static frontend + Functions (contact form → Discord webhook)
  • Self-hosted — NixOS machine "Dedalus", backend via systemd
  • Cloudflare Tunnel — cloudflared → public HTTPS/WSS, no open inbound ports
OS / Infra
  • NixOS 25.11 — declarative, reproducible, flake.nix pinned inputs
  • systemd units — ddl-backend · ddl-cloudflared · ddl-ergo
  • XRDP over LAN — XFCE desktop for maintenance access
